Silex\Application::error

Registers an error handler.

Error handlers are simple callables which take a single Exception as an argument. If a controller throws an exception, an error handler can return a specific response. When an exception occurs, all handlers will be called, until one returns something (a string or a Response object), at which point that will be returned to the client. For this reason you should add logging handlers before output handlers.
